# Сага управления комментарием

## Бизнес-процесс

1. **Создать комментарий** -> статус `NEW`
2. **Оплатить комментарий** -> статус `PAID`
3. **Завершить комментарий** -> статус `DONE`

## Состояния комментария

| Статус | Описание |
|--------|----------|
| `NEW` | Комментарий создан, ожидает оплаты |
| `PAID` | Комментарий оплачен, готов к завершению |
| `DONE` | Комментарий завершён |
| `CANCELLED` | Комментарий отменён (ошибка оплаты) |

+---------------+## Компенсация при ошибке оплаты

Если на шаге оплаты происходит ошибка:
1. Комментарий автоматически переводится в статус `CANCELLED`
2. Система логирует ошибку
3. Повторная оплата невозможна - нужно создавать новый комментарий

## Механизм Retry

При ошибке оплаты система может:
1. Повторить попытку оплаты до 3 раз
2. Если все попытки неудачны - перевести в `CANCELLED`
3. Логировать каждую попытку для аудита
