1. Ресурс vs Коллекция
Ресурс — конкретный объект (сущность). URL: /users/42 (пользователь с ID 42).
Коллекция — список (множество) ресурсов. URL: /users (все пользователи).

2. POST и 201 Created
POST — метод неидемпотентный, создает новый подчиненный ресурс каждый раз при повторе.
201 Created — сигнализирует: «Объект успешно создан, смотри результат в Location». 200 OK просто говорит «Запрос выполнен», что менее информативно для операции создания.

3. Идемпотентность
Смысл: Многократное повторение одного и того же запроса приводит к тому же результату (состоянию сервера), что и однократное выполнение.
Идемпотентные методы: GET, PUT, DELETE, HEAD, OPTIONS. (POST и PATCH — не идемпотентны).

4. Генерация ID на сервере
Для гарантии уникальности и последовательности в распределенной системе.
Исключает коллизии, которые неизбежны при генерации ID на клиенте.

5. Контракт API
Это: Соглашение о форматах запросов и ответов (поля, типы данных, коды ошибок).
Важность: Позволяет фронтенду и бэкенду разрабатываться параллельно и независимо, не ломая логику друг друга.

6. Зачем нужен OpenAPI (Swagger)
Единый источник правды.
Автогенерация: Клиентского кода (SDK), серверных заглушек и интерактивной документации для тестирования без Postman.
