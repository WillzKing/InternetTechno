1. Forward Proxy vs Reverse Proxy
Forward: Скрывает клиента от сервера. Работает на стороне отправителя (обход блокировок, анонимность).
Reverse: Скрывает сервер от клиента. Работает на стороне получателя (балансировка, единая точка входа).

2. Upstream (Nginx)
Группа бэкенд-серверов, куда Nginx перенаправляет запросы. По сути — пул серверов приложений, указанный в блоке upstream {}.

3. Зачем скрывать IP сервисов?
Безопасность: Невозможно атаковать конкретный сервер напрямую в обход файрвола шлюза.
Гибкость: Можно менять инфраструктуру или масштабироваться, не меняя DNS и не уведомляя клиентов.

4. Задачи API Gateway (кроме маршрутизации)
Rate Limiting (ограничение частоты запросов).
Аутентификация (проверка JWT/токенов на входе).

5. Отказ Gateway и защита
Произойдет: Полная недоступность всех сервисов, даже если бэкенды работают идеально (SPOF — единая точка отказа).
Защита:
Кластер Gateways + Keepalived (виртуальный IP).
Несколько инстансов за балансировщиком облака.
